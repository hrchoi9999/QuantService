# QS 요청 접수 템플릿

이 문서는 `QS-Master`가 Quant / QuantMarket / 내부 QS 요청을 접수할 때 사용하는 표준 템플릿입니다.

기본 원칙:

- 외부 요청은 기본적으로 이 템플릿으로 정리합니다.
- Quant / QM에도 이 템플릿 형식을 전달해서 그대로 써도 됩니다.
- `권장 담당 쓰레드`는 요청자가 적어도 되고 비워도 됩니다.
- `대표 쓰레드`, `보조 쓰레드`, `배포 필요 여부`는 `QS-Master`가 최종 확정합니다.

## 외부 전달용 간단 템플릿

Quant / QM에 보낼 때는 아래 형식을 그대로 전달해도 됩니다.

```text
[QS 작업요청]

요청명:
요청 출처: Quant / QuantMarket
QS 요청 제출처: QS-Master
권장 담당 쓰레드:
공개 반영 포함 여부: Yes / No
Admin only 여부: Yes / No
배포 필요 예상: Yes / No / TBD

작업 목적:

필수 참조 문서:
- 

사용 payload / 파일 / API:
- 

QS에서 반영해야 할 내용:
1.
2.
3.

이번 작업에서 반영하지 말아야 할 범위:
- 

fallback / optional 규칙:
- 

QA 체크리스트:
- [ ]
- [ ]
- [ ]
```

## QS-Master 내부 정리용 템플릿

`QS-Master`에서는 접수 후 아래 형식으로 최종 정리합니다.

```text
[QS-Master Intake]

요청명:
접수일:
요청 출처:

원문 문서:
- 

대표 쓰레드:
보조 쓰레드:

공개 반영 여부:
Admin only 여부:
배포 필요 여부:

핵심 목적:

QS 작업 범위:
1.
2.
3.

비범위:
- 

리스크 / 경계:
- 공개/비공개 경계:
- 컴플라이언스:
- 권한/보안:
- 데이터 의존:

필수 검증:
- pytest:
- ruff:
- black --check:
- live 확인:

외부 후속 요청 필요 여부:
- Quant:
- QM:

상태:
- 접수
- 배정 완료
- 구현 중
- 검증 완료
- 배포 완료
```

## 사용 규칙

1. Quant / QM에는 `외부 전달용 간단 템플릿`을 보냅니다.
2. 실제 작업 시작 전 `QS-Master`는 내부 정리용 템플릿으로 다시 정리합니다.
3. 대표 쓰레드가 정해지면 해당 쓰레드 `WORKLOG.md`와 `BACKLOG.md`에도 연결합니다.
4. 공개 반영 작업은 반드시 `RELEASE_GATE.md` 기준으로 마감합니다.
