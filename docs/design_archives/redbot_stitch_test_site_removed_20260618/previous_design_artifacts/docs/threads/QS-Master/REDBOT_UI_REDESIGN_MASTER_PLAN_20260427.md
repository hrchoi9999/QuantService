# REDBOT 웹 UI 개편 전체 작업계획서 (Master)

- 작성일: 2026-04-27
- 작성 스레드: QS-Master
- 대상 서비스: redbot.co.kr (Public/User/Admin)
- 관련 시스템: QuantMarket, Quant, QuantService

---

## 1) 작업 목적

redbot.co.kr의 정보 밀도가 높은 화면을 **콘텐츠 변경 없이** UI/UX 중심으로 개편하여,

1. 직관성 및 가독성 향상
2. 금융 서비스에 맞는 전문적 인상 강화
3. 라이트 기본 + 다크 선택형 테마 체계 확립
4. 사용자 화면과 관리자 화면의 목적/권한 분리 강화

를 달성한다.

---

## 2) 절대 원칙 (필수 준수)

1. 기본 화면은 밝고 화사한 톤(라이트 모드)으로 운영한다.
2. 사용자 선택으로 다크 모드 전환 가능해야 한다.
3. 기존 정보 구조/콘텐츠/계산 로직은 유지하고, 배치/폰트/톤/컴포넌트 중심으로 개선한다.
4. redbot 로고 색상은 유지하고, 나머지 색상 시스템은 원점 재설계한다.
5. 사용자용 화면과 관리자용 화면은 IA/권한/UI 모두 분리하여 운영한다.
6. 회원가입/로그인/투자현황 등 입력형 페이지의 가독성과 안정감을 우선 개선한다.

---

## 3) 운영 안전 원칙 (배포 게이트)

> **검증 완료 전까지 기존 서비스에 지장이 없도록 유지하고, 검증이 완료된 후에만 배포한다.**

세부 적용:

1. 개편 작업은 기본적으로 feature-flag 또는 경로 분리(또는 staging 전용 반영)로 진행한다.
2. 기존 public 라우트/기능은 검증 완료 전 절대 교체하지 않는다.
3. QA 체크리스트와 회귀 테스트 통과 후에만 release gate 승인한다.
4. 배포는 단계 배포(canary/점진 반영) 원칙을 적용한다.
5. 이상 징후 발생 시 즉시 롤백 가능 상태(직전 안정 이미지/설정) 유지한다.

---

## 4) 범위 정의

### 4.1 포함 범위

- Public 핵심 화면: 홈, 시장 브리핑, 전략별 퀀트모델, 성과 설명, 변경내역, 상승종목 발굴
- User 화면: 로그인/회원가입/투자현황
- Admin 화면: 관리자 전용 대시보드/분석 화면(기능 유지 + 표현 개선)
- 공통 디자인 시스템: 색상/타이포/간격/카드/테이블/탭/폼/배지/차트

### 4.2 제외 범위

- Quant/QuantMarket의 계산 로직 변경
- 투자자문성 문구/기능 추가
- 인증/권한 정책의 본질 변경(단, UI/노출 정책 정리는 포함)

---

## 5) 데이터/차트 연동 전략

## 5.1 1차 구현 원칙

- **QS는 차트 렌더러/UX를 구현하고, 데이터 생산은 도메인 소유 스레드에 요청한다.**
- QS가 시장/모델 계산 로직을 중복 구현하지 않는다.

## 5.2 역할 분담

1. QuantMarket
   - 시장 브리핑 계열 history payload 제공
   - 예: timeline, asset_strength, state_transition, next_day_preview(overnight assets)
2. Quant 모델 쓰레드
   - 모델/종목/성과 history payload 제공
   - 예: user/internal/tseries performance & holdings/change history
3. QS-Public-Web
   - 차트 UI(ECharts) 구현
   - 라이트/다크 테마 동기화
   - empty/stale/fallback UX
4. QS-QM-Handoff
   - payload schema 계약/단위/시간대/as_of_date 정합성 관리

---

## 6) 단계별 실행계획

## Phase 0. 기준선 확정 (D+0 ~ D+2)

- 화면/컴포넌트 인벤토리 작성
- 페이지별 정보 우선순위(요약 > 상세) 재정의
- 디자인 토큰 초안(라이트/다크) 확정
- 배포 안전 계획(feature-flag/경로분리) 확정

산출물:
- UI 인벤토리 문서
- 토큰 초안
- 배포/롤백 체크리스트

## Phase 1. 공통 디자인 시스템 구축 (D+2 ~ D+6)

- 타이포 체계(헤드라인/본문/라벨/숫자) 확정
- 공통 컴포넌트(카드/테이블/탭/배지/폼) 통일
- 차트 wrapper 및 포맷 유틸 구현

산출물:
- 공통 CSS/컴포넌트 규격
- 테마 토글 구현

## Phase 2. 데이터 계약 고정 (병행, D+2 ~ D+8)

- QuantMarket/Quant에 history payload 계약 요청
- schema/필수 필드/source_name/as_of_date/generated_at/freshness 확정
- stale/fallback 정책 확정

산출물:
- payload 계약서
- API/loader 매핑표

## Phase 3. Public 핵심 화면 반영 (D+6 ~ D+12)

- 홈/시장브리핑/전략별퀀트모델 우선
- 시각 계층 및 그래프 반영
- 기존 콘텐츠/수치 유지 검증

산출물:
- Public 개편 1차
- 비교 캡처(개편 전/후)

## Phase 4. User/Admin 화면 개선 (D+10 ~ D+15)

- 투자현황/로그인/회원가입 폼 가독성 강화
- admin 전용 화면의 정보밀도/필터 중심 UX 정리
- 사용자/관리자 UI 분리 일관성 점검

산출물:
- User/Admin 개편 1차

## Phase 5. 통합 검증 및 배포 (D+14 ~ D+18)

- 기능/회귀/권한/반응형/접근성 점검
- release gate 승인
- 점진 배포 후 모니터링

산출물:
- QA 보고서
- 배포 보고서

---

## 7) QA/검증 체크리스트 (핵심)

1. 기존 기능/데이터 정확성 유지 여부
2. 라이트/다크 전환 시 가독성/대비 정상 여부
3. 시장/모델 차트 수치와 payload 정합성
4. stale/누락 데이터 시 graceful fallback 동작
5. 모바일/태블릿/데스크톱 반응형 정상
6. 권한별 메뉴/페이지 접근 분리 정상
7. 로그인/회원가입/투자현황 입력 UX 안정성
8. public 페이지에서 비자문 표현 원칙 준수

---

## 8) 배포 정책

1. Pre-Prod 검증 완료 전 본 운영 교체 금지
2. `pytest/ruff/black --check` + 수동 화면검증 + live smoke check 통과 필수
3. 1차 배포는 영향 범위 작은 화면부터 순차 배포
4. 배포 후 지정 모니터링 창구에서 이상 탐지 시 즉시 롤백
5. 롤백 후 원인/재발방지 문서화

---

## 9) 리스크 및 대응

1. 데이터 지연/누락
   - 대응: 캐시 상태/신선도 배지 + fallback 메시지 + 마지막 정상 스냅샷 표시
2. 지표 해석 혼선(CAGR vs 1Y 수익률 등)
   - 대응: metric label + basis 라벨 명시
3. 권한 노출 이슈
   - 대응: server-side 권한 체크 우선, UI 숨김은 보조 수단
4. UI 대규모 변경에 따른 회귀
   - 대응: 페이지 단위 단계 배포 + 체크리스트 기반 회귀 테스트

---

## 10) 완료 정의 (Definition of Done)

1. 기존 정보 제공 내용은 유지된 상태에서 UI 가독성 개선 완료
2. 라이트 기본/다크 선택형 운영 안정화
3. 핵심 페이지에 차트 기반 인사이트가 정확히 반영
4. 사용자/관리자 화면 분리 운영 일관성 확보
5. 검증 완료 후에만 배포 원칙 준수 및 배포 보고 완료
