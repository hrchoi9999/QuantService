# Review TASK 03

## 우선순위
- P1
- 범주: 보안 / 관리자 접근제어

## 핵심 이슈
관리자 접근 키가 URL query string(`access_key`)로 허용되고, 내부 관리자 링크도 이를 그대로 전파할 수 있습니다.

- 근거 코드
  - `service_platform/feedback/handlers.py:25-37`
  - `service_platform/web/app.py:684-707`

## 왜 문제인가
- URL에 실린 관리자 키는 브라우저 히스토리, 서버 로그, 리퍼러 헤더, 캡처 화면으로 쉽게 새어 나갑니다.
- 링크 공유나 북마크만으로도 장기 노출 사고가 날 수 있습니다.
- 지금은 로그인 관리자면 정상 접근 가능하므로, query-string 백도어를 유지할 실익이 작습니다.

## 작업 목표
관리자 인증을 세션 기반으로 일원화하고, 예외적으로 필요한 경우에도 헤더 기반 단기 키만 허용합니다.

## 작업 지시
1. `is_admin_request()`에서 `request.args.get("access_key")` 허용을 제거합니다.
2. `admin_url()`과 `build_admin_links()`에서 query string으로 키를 붙이는 로직을 제거합니다.
3. 관리자 접근 정책을 다음 중 하나로 정리합니다.
   - 기본: 로그인 + admin role
   - 예외 운영용: `X-Admin-Key` 헤더만 허용
4. 관련 테스트를 갱신합니다.
5. 운영 문서에 “URL로 관리자 키를 전달하지 않는다”는 규칙을 명시합니다.

## 필수 테스트
- 로그인한 admin role 사용자는 기존처럼 접근 가능
- `?access_key=...` 만으로는 접근 불가
- 필요 시 `X-Admin-Key` 헤더 기반 접근만 허용

## 완료 기준
- 관리자 키가 URL에 노출되지 않습니다.
- 관리자 내비게이션 링크에 민감정보가 포함되지 않습니다.
- 접근제어 테스트가 새 정책 기준으로 갱신됩니다.
