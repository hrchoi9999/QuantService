# Review TASK 04

## 우선순위
- P1
- 범주: 보안 / 외부 링크 처리 / 피싱 리스크

## 핵심 이슈
클릭 트래킹 엔드포인트가 `target` 쿼리 파라미터를 검증 없이 그대로 redirect 합니다.

- 근거 코드
  - `service_platform/web/app.py:1427-1440`

## 왜 문제인가
- `https://redbot.co.kr/e/click?...&target=https://malicious.example` 형태의 URL이 그대로 동작할 수 있습니다.
- 사용자는 `redbot.co.kr` 도메인을 신뢰하고 클릭하기 때문에 피싱/우회 링크로 악용되기 쉽습니다.
- 분석 이벤트에는 도움이 되지만, 보안상 외부 임의 URL 허용은 위험합니다.

## 작업 목표
트래킹 링크 목적지를 화이트리스트 기반으로 제한하고, 기본 목적지는 서버에서만 생성합니다.

## 작업 지시
1. `target` 사용자 입력을 직접 redirect 하지 않도록 변경합니다.
2. 가능한 방식
   - 권장: `ticker` 기준으로 서버가 목적지 URL을 계산
   - 대안: 허용 도메인 화이트리스트(`finance.naver.com` 등) 검증 후에만 redirect
3. 허용되지 않은 목적지는 400 또는 안전한 기본 페이지(`/today`)로 보냅니다.
4. 분석 이벤트에는 최종 허용된 URL만 기록합니다.
5. 테스트를 추가합니다.

## 필수 테스트
- 정상 ticker 링크는 기존처럼 이동
- 외부 임의 도메인 `target`은 차단
- 허용 도메인만 통과하도록 검증

## 완료 기준
- `e/click`이 오픈 리다이렉트로 동작하지 않습니다.
- 외부 링크 목적지는 서버 통제 하에 있습니다.
- 링크 보안 회귀 테스트가 추가됩니다.
