# QuantService 리뷰 작업지시문

아래 순서대로 작업해 주세요. 운영 리스크가 큰 항목부터 먼저 처리하는 기준입니다.

## 우선순위 순서
1. `Review_TASK_01.md`
2. `Review_TASK_02.md`
3. `Review_TASK_03.md`
4. `Review_TASK_04.md`
5. `Review_TASK_05.md`

## 작업 원칙
- P0 항목은 UI 개선보다 먼저 처리합니다.
- 각 TASK는 코드 수정 + 테스트 추가 + 회귀 확인까지 포함합니다.
- 조회(read) 동작이 상태(write)를 바꾸는 구조는 우선 제거합니다.
- 인증/결제/관리자 기능은 “편의”보다 “안전한 기본값”을 우선합니다.

## 제출 형식
각 TASK별로 아래 내용을 남겨 주세요.

1. 변경한 파일 목록
2. 수정 전 문제 요약
3. 수정 내용
4. 추가/수정한 테스트
5. 남아 있는 리스크 또는 후속 과제

## 이번 리뷰의 핵심 메시지
- 가장 급한 것은 회원 인증 상태 오염 방지와 공개 POST 보안 보강입니다.
- 그 다음은 관리자 접근키 노출 제거와 오픈 리다이렉트 차단입니다.
- UI는 이미 방향성은 잡혀 있으므로, 기능 안정화 이후 구조 리팩터링으로 들어가는 것이 좋습니다.
